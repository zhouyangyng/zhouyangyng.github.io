解决Xcode9打包上传失败的问题

发表于 2017-09-26

问题描述：

前几天刚升级的Xcode9，今天第一次使用Xcode9打包更新APP的版本，结果上传ipa包的时候报错了:

去网上查找了方法，很多人说是cocoapods的原因，解决方法是：在项目的podfile文件中，添加下面这段代码(代码中”TargetName”换成自己的)：

post_install do |installer|
copy_pods_resources_path = “Pods/Target Support Files/Pods-*TargetName/Pods-TargetName-resources.sh”
string_to_replace = ‘–compile “${BUILT_PRODUCTS_DIR}/${UNLOCALIZED_RESOURCES_FOLDER_PATH}”‘
assets_compile_with_app_icon_arguments = ‘–compile “${BUILT_PRODUCTS_DIR}/${UNLOCALIZED_RESOURCES_FOLDER_PATH}” –app-icon “${ASSETCATALOG_COMPILER_APPICON_NAME}” –output-partial-info-plist “${BUILD_DIR}/assetcatalog_generated_info.plist”‘
text = File.read(copy_pods_resources_path)
new_contents = text.gsub(string_to_replace, assets_compile_with_app_icon_arguments)
File.open(copy_pods_resources_path, “w”) {|file| file.puts new_contents }
end

接着 pod install 或者 pod update

我试了这个方法之后，依然没有解决，虽然ipa传上去了，但是传完后报错是说缺少了1024px的APP大图标，然后ipa无效。

后来在博客https://yohunl.com/xcode9da-bao-shang-chuan-yu-dao-de-wen-ti-he-jie-jue-fang-shi/ 中看到另一种方法，在编译结束时，找到对应的.xarchive文件，具体方法参考该博客。最后使用终端输入命令：

find Products/ -name Info.plist -print0 | xargs -0n1 plutil -replace BuildMachineOSBuild -string 16A323

博主说问题解决了，可是我的就是没用啊！😂 试了很多次，网上也找了很多文章，我的ipa一次次被拒绝。

思考：

1024px的图标是iOS11之后才有的，使用Xcode9上传时是不能缺少的，但是我的项目中确实有，并且图片也没有任何问题。于是打包后，查看了包内容，看看这个神奇的icon到底有没有？
查看的方法如下：

  1. 编译后，对archive文件右击，然后 Show In Finder。

  2. archive文件右击 -> 显示包内容，找到/Products/Applications/xxx.app，这就是打完包的app。

  3. 接着右击 -> 显示包内容，就可以看到所有的文件了。往下翻，看到了所有的APP图标，明明项目中一共18个icon，这里居然17个，1024px的到底哪去了？😨 或许这是Xcode9.0的bug吧。

我的解决方法：

看到苹果官网的论坛上面讨论这个问题https://forums.developer.apple.com/thread/86290 ，仔细的看完了下面所有的回复，最终找到一个方法。
还是刚刚的路径，在 xxx.app 显示包内容后，找到 info.plist 文件，该文件用于记录项目打包后的配置信息。打开info.plist，把其中的几个属性值修改成下面的结果：

  1. DTXcode -> “0833”
  2. DTSDKName -> “iphoneos10.3”
  3. DTSDKBuild -> “14E8301”
  4. DTPlatformName -> “iphoneos”
  5. DTCompiler -> “com.apple.compilers.llvm.clang.1_0”
  6. DTXcodeBuild -> “8E3004b”
  7. DTPlatformVersion -> “10.3”
  8. DTPlatformBuild -> “14E8301”

修改之后，导出ipa包，再次上传，成功！其实这种方法，就是把配置信息修改成Xcode8，虽然使用的是Xcode9，但是APP Store会认为是Xcode8, 因此APP Store不会验证1024px的icon，这样就解决了图标丢失的问题了，当然还要记得去iTunes Connect把APP的icon换成1024px的。
